DATA

TCGA-BRCA stands for The Cancer Genome Atlas Breast Invasive Carcinoma. 
It is a comprehensive, multi-platform dataset containing molecular and clinical data from over 1,000 breast cancer patients. 

As part of the broader The Cancer Genome Atlas (TCGA) program, its goal is to map the genomic changes in breast cancer to help researchers understand the disease's biology and develop better treatments.

The files are split into two main categories: Biospecimen (physical samples) and Clinical (patient history).

Here is a breakdown of what each file typically contains:

1. Biospecimen Data
These files track the "genealogy" of the biological sample from the patient to the lab.

sample.tsv: The most critical file here. it identifies if a sample is Tumor (Primary or Metastatic) or Normal (Solid Tissue or Blood).
slide.tsv: Links the physical tissue samples to the high-resolution diagnostic or frozen section imaging slides.
portion.tsv, analyte.tsv, aliquot.tsv: These describe the technical processing steps. An aliquot is the final "unit" of DNA/RNA used for sequencing.

2. Clinical Data
These files provide the medical context for the genomic data.

clinical.tsv: The primary file. It contains core patient data like age at diagnosis, gender, vital status (alive/dead), and days to death or last follow-up.
pathology_detail.tsv: Contains technical medical details like Tumor Stage (TNM system), histologic grade, and specific breast cancer markers (e.g., ER, PR, and HER2 status).
follow_up.tsv: Tracks the patient over time, documenting any new tumor events or subsequent treatments.
exposure.tsv / family_history.tsv: Details on environmental risk factors (like smoking history) and whether the patient has a documented family history of cancer.

3. Annotations Table

annotations-table...tsv: This is a quality control file. It lists any known issues with specific samples (e.g., "sample contaminated" or "patient withdrew consent"). Always check this to see if you should exclude certain samples from your analysis.

How to use them together
To analyze this data, you typically use a unique identifier called the Case ID (e.g., TCGA-A8-A07S) to merge these tables. For example, you would link the survival data in clinical.tsv to the tumor type in sample.tsv to see which subtypes have better outcomes.


follow_up.tsv is your most valuable file — and it has two distinct layers inside it. The follow_ups.* columns give you longitudinal disease response, recurrence timing, and ECOG performance status. But buried inside it are molecular_tests.* columns — gene symbols, test results, variant types. This is the actual molecular biomarker landscape. These are not pre-specified trial markers, they're mutation-level data: PIK3CA, TP53, BRCA1 status per patient. This is exactly what BSE's discovery engine should be clustering on.

pathology_detail.tsv has the tumor biology features that almost never appear in trial protocols: tumor-infiltrating lymphocytes (tumor_infiltrating_lymphocytes), vascular invasion (vascular_invasion_present), necrosis percentage (necrosis_percent), perineural invasion. These are continuous histological biomarkers — strong candidates for discovering subpopulations that trials miss.

clinical.tsv gives you staging and demographics. The treatments.* block already in there gives you treatment_outcome and therapeutic_agents — so you can stratify outcome by drug.

sample.tsv is mostly for QC — tissue type and preservation method are batch effect covariates you should account for, not discovery targets.
